### 1.2 集合命名规范

| 规范 | 说明 |
|------|------|
| 命名风格 | 小写蛇形命名法（snake_case） |
| 集合名 | 复数形式，表示一组数据（如 `users` 而非 `user`） |
| 关联表 | 使用下划线连接关联关系（如 `task_claims`、`agreement_signatures`） |
| 保留字 | `_id`（主键）、`_openid`（用户标识）为微信云开发系统保留字段 |

### 1.3 数据类型说明

| 类型标识 | 对应云数据库类型 | 说明 |
|----------|:--------------:|------|
| `string` | String | 字符串 |
| `number` | Number | 数值（整数或浮点数） |
| `integer` | Number(整数) | 整数值，用于枚举状态 |
| `boolean` | Boolean | 布尔值 |
| `date` | Date | 日期时间（ISODate） |
| `array` | Array | 数组 |

### 2.1 `users` — 用户信息表

> **用途**：存储用户基本信息、实名认证状态、活体认证有效期。  
> **权限**：`_openid` 隔离，用户仅可读写本人数据。  
> **关联**：← `task_claims._openid`，← `agreement_signatures._openid`，← `settlements._openid`

| # | 字段名 | 类型 | 必填 | 默认值 | 中文注释 |
|---|--------|------|:----:|--------|----------|
| 1 | `_id` | string | 是 | 自动 | 用户记录ID（云数据库自动生成） |
| 2 | `_openid` | string | 是 | 自动 | 微信用户唯一标识（小程序端自动注入，不可伪造） |
| 3 | `name` | string | 是 | — | 用户真实姓名（用于四要素认证） |
| 4 | `idCard` | string | 是 | — | 身份证号码，18位（含末位X），用于四要素认证 |
| 5 | `bankCard` | string | 是 | — | 银行卡号，16-19位数字，用于收款绑定 |
| 6 | `phone` | string | 是 | — | 手机号码，11位数字，用于通信及四要素认证 |
| 7 | `bankName` | string | 否 | `null` | 开户银行名称（如：中国工商银行），用户可自行填写 |
| 8 | `idCardFrontUrl` | string | 否 | `null` | 身份证正面（人像面）照片云存储文件ID |
| 9 | `idCardBackUrl` | string | 否 | `null` | 身份证反面（国徽面）照片云存储文件ID |
| 10 | `identityVerified` | boolean | 是 | `false` | 四要素实名认证是否已通过（true=已认证 / false=未认证） |
| 11 | `identityTime` | date | 否 | `null` | 最近一次四要素认证通过的时间 |
| 12 | `livenessVerified` | boolean | 是 | `false` | 活体人脸认证是否已通过（true=已认证 / false=未认证） |
| 13 | `livenessDate` | date | 否 | `null` | 最近一次活体认证通过的时间 |
| 14 | `livenessExpireDate` | date | 否 | `null` | 活体认证过期时间（认证通过日 + 180个自然日） |
| 15 | `agreementSigned` | boolean | 是 | `false` | 服务协议是否已签署（true=已签 / false=未签），用于快速状态判断 |
| 16 | `createTime` | date | 是 | 当前时间 | 用户首次注册/创建记录的时间 |
| 17 | `updateTime` | date | 是 | 当前时间 | 最近一次资料更新的时间 |

**索引建议**：

| 索引字段 | 类型 | 说明 |
|----------|------|------|
| `_openid` | 唯一索引 | 系统自动创建，主查询条件 |
| `identityVerified` + `livenessVerified` | 复合索引 | 认证状态筛选 |

**业务规则**：
- 每个微信用户（通过 `_openid`）最多有 1 条记录
- 活体认证通过后，`livenessExpireDate` = 认证当天 + 180天
- `agreementSigned` 为冗余字段，用于快速判断；详细签署记录见 `agreement_signatures` 表
- 前端展示时需做脱敏处理（姓名脱敏、身份证脱敏、手机号脱敏、银行卡脱敏）

**示例数据**：
```json
{

  "_openid": "oABC123xyz456",
  "name": "张三",
  "idCard": "110101199001011234",
  "bankCard": "6228480012345678912",
  "phone": "13800138000",
  "bankName": "中国工商银行",
  "idCardFrontUrl": "cloud://env-id.xxx/images/idcard_front.jpg",
  "idCardBackUrl": "cloud://env-id.xxx/images/idcard_back.jpg",
  "identityVerified": true,
  "identityTime": "2026-06-01T10:30:00.000Z",
  "livenessVerified": true,
  "livenessDate": "2026-06-01T10:35:00.000Z",
  "livenessExpireDate": "2026-11-28T10:35:00.000Z",
  "agreementSigned": true,
  "createTime": "2026-06-01T10:00:00.000Z",
  "updateTime": "2026-06-01T10:35:00.000Z"
}
```

### 2.2 `tasks` — 任务表

> **用途**：存储运营发布的所有任务，是任务大厅的数据源。  
> **权限**：所有用户可读，仅后台可写。  
> **关联**：→ `task_claims.taskId`

| # | 字段名 | 类型 | 必填 | 默认值 | 中文注释 |
|---|--------|------|:----:|--------|----------|
| 1 | `_id` | string | 是 | 自动 | 任务ID（云数据库自动生成） |
| 2 | `title` | string | 是 | — | 任务名称/标题（展示在任务大厅卡片上） |
| 3 | `description` | string | 是 | — | 任务详细描述（展示在任务详情页） |
| 4 | `deliveryStandard` | string | 是 | — | 交付标准说明，描述完成此任务需达到的质量要求 |
| 5 | `enterpriseName` | string | 是 | — | 发布该任务的企业名称 |
| 6 | `payMode` | integer | 是 | — | 付费模式：1=按单价计酬，2=按比例计酬 |
| 7 | `unitPrice` | number | 否 | `null` | 单价金额（元），仅当 payMode=1 时有值 |
| 8 | `commissionRate` | number | 否 | `null` | 佣金比例（百分比，如15表示15%），仅当 payMode=2 时有值 |
| 9 | `totalQuota` | integer | 是 | — | 任务总名额（可领取的总人数上限） |
| 10 | `claimedQuota` | integer | 是 | `0` | 已被领取的名额数（每次领取+1，原子递增） |
| 11 | `status` | integer | 是 | `0` | 任务状态：0=未发布（运营草稿），1=进行中（大厅可见），2=已结束 |
| 12 | `startDate` | date | 是 | — | 任务开放领取的开始时间 |
| 13 | `endDate` | date | 是 | — | 任务领取截止时间 |
| 14 | `requirements` | array | 否 | `[]` | 硬性条件标签列表（如"年满18周岁""完成实名认证"），展示在任务卡片和详情页 |
| 15 | `requirementDescription` | string | 否 | `""` | 硬性条件详细说明文案，展示在任务详情页硬性要求区域的描述 |
| 16 | `coverImage` | string | 否 | `null` | 任务封面图云存储文件ID |
| 17 | `createTime` | date | 是 | 当前时间 | 任务创建时间 |
| 18 | `updateTime` | date | 是 | 当前时间 | 任务最近更新时间 |

**索引建议**：

| 索引字段 | 类型 | 说明 |
|----------|------|------|
| `status` + `startDate` + `endDate` | 复合索引 | 任务大厅查询：status=1 且时间范围内 |
| `createTime` | 降序索引 | 按创建时间倒序排列 |

**枚举值定义**：

| 字段 | 值 | 说明 |
|------|----|------|
| `payMode` | `1` | 单价模式 — 按固定单价计酬 |
| | `2` | 比例模式 — 按交易额百分比计酬 |
| `status` | `0` | 未发布 — 运营侧创建的草稿，大厅不可见 |
| | `1` | 进行中 — 在有效期内且未满额，大厅可见 |
| | `2` | 已结束 — 超过截止时间或名额已满 |

**业务规则**：
- 任务大厅只展示 `status=1` 且 `当前时间 ∈ [startDate, endDate]` 的任务
- 剩余名额 = `totalQuota - claimedQuota`，需 ≤0 时禁止领取
- `claimedQuota` 增加需使用 `db.command.inc(1)` 防止并发超发
- `requirements` 为字符串数组，每条为一个硬性条件标签，在任务大厅卡片和详情页展示
- `requirementDescription` 为硬性条件的详细说明文案，仅在详情页展开

**示例数据**：
```json
{
  "_id": "auto_generated_id",
  "title": "企业年报数据录入",
  "description": "根据提供的企业工商年报PDF文件，将关键财务数据录入到指定Excel模板中。需确保数据准确无误，格式规范统一。",
  "deliveryStandard": "按要求完成所有数据录入，经审核无误后即为完成。如有错误需在1个工作日内修正。",
  "enterpriseName": "深圳腾讯科技有限公司",
  "payMode": 1,
  "unitPrice": 50.00,
  "commissionRate": null,
  "totalQuota": 100,
  "claimedQuota": 23,
  "status": 1,
  "startDate": "2026-05-01T00:00:00.000Z",
  "endDate": "2026-12-31T23:59:59.000Z",
  "requirements": ["年满18周岁", "完成四要素实名认证", "具备基本Excel操作能力", "每日至少完成5份年报录入"],
  "requirementDescription": "需自行准备联网电脑，使用平台提供的Excel模板进行数据录入。每日至少完成5份年报（约2小时工作量），数据错误率须控制在0.1%以内。未达标将扣除当日任务报酬。",
  "coverImage": null,
  "createTime": "2026-06-01T08:00:00.000Z",
  "updateTime": "2026-06-10T10:00:00.000Z"
}
```

### 2.3 `task_claims` — 任务领取记录表（核心流转表）

> **用途**：记录每次用户领取任务的全生命周期，是任务流转的核心表。  
> **权限**：`_openid` 隔离读，用户可更新自己的交付物和状态。  
> **关联**：`taskId` → `tasks._id`，`_openid` → `users._openid`，← `settlements.claimId`

| # | 字段名 | 类型 | 必填 | 默认值 | 中文注释 |
|---|--------|------|:----:|--------|----------|
| 1 | `_id` | string | 是 | 自动 | 领取记录ID（云数据库自动生成） |
| 2 | `_openid` | string | 是 | 自动 | 领取用户的微信openid |
| 3 | `taskId` | string | 是 | — | 关联任务的ID，对应 `tasks._id` |
| 4 | `taskTitle` | string | 是 | — | 任务标题（冗余存储，避免频繁联表查询） |
| 5 | `reward` | number | 是 | — | 该次领取的预估报酬金额（元），领取时快照 |
| 6 | `status` | integer | 是 | `0` | 领取状态：0=审核中，1=进行中，2=已完成，3=已结算，4=审核拒绝 |
| 7 | `identityStatus` | integer | 是 | `0` | 领取时四要素认证快照：0=未认证，1=已认证 |
| 8 | `livenessStatus` | integer | 是 | `0` | 领取时活体认证快照：0=未认证，1=已认证，2=已过期 |
| 9 | `agreementStatus` | integer | 是 | `0` | 领取时协议签署快照：0=未签署，1=已签署 |
| 10 | `deliveryUrls` | array | 否 | `[]` | 交付物图片URL数组（云存储文件ID列表），用户完成时上传 |
| 11 | `auditRemark` | string | 否 | `null` | 运营审核备注，拒绝时必须填写拒绝原因 |
| 12 | `submitTime` | date | 否 | `null` | 用户提交领取申请的时间 |
| 13 | `auditTime` | date | 否 | `null` | 运营审核操作的时间 |
| 14 | `completeTime` | date | 否 | `null` | 用户上传交付物、标记完成的时间 |
| 15 | `settleTime` | date | 否 | `null` | 结算完成的时间，由结算系统写入 |
| 16 | `createTime` | date | 是 | 当前时间 | 领取记录创建时间 |
| 17 | `updateTime` | date | 是 | 当前时间 | 记录最近更新时间 |

**索引建议**：

| 索引字段 | 类型 | 说明 |
|----------|------|------|
| `_openid` + `status` | 复合索引 | 我的任务列表查询，按状态筛选 |
| `taskId` + `_openid` | 复合唯一索引 | 防止同一用户重复领取同一任务 |
| `status` | 普通索引 | 后台按状态筛选 |

**状态流转图**：
```
                      ┌──────────┐
                      │ 审核中(0) │  ← 用户提交领取申请
                      └────┬─────┘
                 ┌─────────┼─────────┐
                 │运营审核           │运营审核
                 │通过              │拒绝
                 ▼                  ▼
          ┌──────────┐      ┌───────────┐
          │ 进行中(1) │      │审核拒绝(4) │
          └────┬─────┘      └─────┬─────┘
               │用户上传          │用户重新提交
               │交付物            │(status→0)
               ▼                  │
          ┌──────────┐            │
          │ 已完成(2) │◄───────────┘
          └────┬─────┘
               │运营/系统结算
               ▼
          ┌──────────┐
          │ 已结算(3) │
          └──────────┘
```

**枚举值定义**：

| 字段 | 值 | 中文含义 | 说明 |
|------|----|----------|------|
| `status` | `0` | 审核中 | 已提交，等待运营审核 |
| | `1` | 进行中 | 审核通过，用户执行任务中 |
| | `2` | 已完成 | 交付物已上传，待结算 |
| | `3` | 已结算 | 报酬已结算到账 |
| | `4` | 审核拒绝 | 审核不通过，可修正后重新提交（状态→0） |
| `identityStatus` | `0` | 未认证 | — |
| | `1` | 已认证 | — |
| `livenessStatus` | `0` | 未认证 | — |
| | `1` | 已认证 | — |
| | `2` | 已过期 | 活体认证已超过180天有效期 |
| `agreementStatus` | `0` | 未签署 | — |
| | `1` | 已签署 | — |

**业务规则**：
- 同一用户 + 同一任务 + 非拒绝状态，不可重复领取（唯一约束）
- 认证快照字段（identityStatus/livenessStatus/agreementStatus）在领取时固化，后续用户重新认证不会自动回溯更新
- `reward` 在领取时快照计算；单价模式 = unitPrice，比例模式暂存0（实际报酬由后台验收后确定）
- 状态 `4→0` 重新提交时，`submitTime` 更新为当前时间，`auditRemark` 清空

**示例数据**：
```json
{
  "_id": "auto_generated_claim_id",
  "_openid": "oABC123xyz456",
  "taskId": "task_auto_id_001",
  "taskTitle": "企业年报数据录入",
  "reward": 50.00,
  "status": 1,
  "identityStatus": 1,
  "livenessStatus": 1,
  "agreementStatus": 1,
  "deliveryUrls": [],
  "auditRemark": null,
  "submitTime": "2026-06-10T14:00:00.000Z",
  "auditTime": "2026-06-10T15:00:00.000Z",
  "completeTime": null,
  "settleTime": null,
  "createTime": "2026-06-10T14:00:00.000Z",
  "updateTime": "2026-06-10T15:00:00.000Z"
}
```

### 2.4 `identity_records` — 四要素认证记录表

> **用途**：记录每次四要素认证的详细信息，便于审计和对账。  
> **权限**：`_openid` 隔离读写。  
> **关联**：`_openid` → `users._openid`

| # | 字段名 | 类型 | 必填 | 默认值 | 中文注释 |
|---|--------|------|:----:|--------|----------|
| 1 | `_id` | string | 是 | 自动 | 认证记录ID（云数据库自动生成） |
| 2 | `_openid` | string | 是 | 自动 | 认证用户的微信openid |
| 3 | `name` | string | 是 | — | 认证时提交的真实姓名 |
| 4 | `idCard` | string | 是 | — | 认证时提交的身份证号码（18位） |
| 5 | `bankCard` | string | 是 | — | 认证时提交的银行卡号 |
| 6 | `phone` | string | 是 | — | 认证时提交的手机号 |
| 7 | `idCardFrontUrl` | string | 是 | — | 认证时上传的身份证正面照云存储文件ID |
| 8 | `idCardBackUrl` | string | 是 | — | 认证时上传的身份证反面照云存储文件ID |
| 9 | `verifyResult` | boolean | 是 | — | 第三方实名认证结果（true=通过 / false=未通过） |
| 10 | `verifyMessage` | string | 否 | `null` | 第三方认证服务返回的描述信息（如"姓名与证件号不匹配"） |
| 11 | `thirdPartyOrderId` | string | 否 | `null` | 第三方认证服务返回的订单号/流水号，用于对账 |
| 12 | `createTime` | date | 是 | 当前时间 | 本次认证请求发起的时间 |

**索引建议**：

| 索引字段 | 类型 | 说明 |
|----------|------|------|
| `_openid` + `createTime` | 复合降序索引 | 按用户查询认证历史 |

**业务规则**：
- 每次提交认证（无论成功或失败）都新增一条记录
- `verifyResult=false` 的记录用于分析认证失败原因，优化用户体验
- 该表为审计日志表，数据不用于业务逻辑判断（业务状态看 `users.identityVerified`）

### 2.5 `liveness_records` — 活体认证记录表

> **用途**：记录每次活体认证的详细信息，区分流程内认证与单独认证。  
> **权限**：`_openid` 隔离读写。  
> **关联**：`_openid` → `users._openid`

| # | 字段名 | 类型 | 必填 | 默认值 | 中文注释 |
|---|--------|------|:----:|--------|----------|
| 1 | `_id` | string | 是 | 自动 | 认证记录ID（云数据库自动生成） |
| 2 | `_openid` | string | 是 | 自动 | 认证用户的微信openid |
| 3 | `verifyResult` | boolean | 是 | — | 腾讯人脸核身结果（true=通过 / false=未通过） |
| 4 | `verifyMessage` | string | 否 | `null` | 人脸核身返回的描述信息（如"光线不足"、"人脸不匹配"） |
| 5 | `sessionId` | string | 否 | `null` | 腾讯人脸核身会话标识，用于 API 调用追溯 |
| 6 | `expireDate` | date | 是 | — | 本次认证的过期时间（认证日 + 180个自然日） |
| 7 | `source` | string | 否 | `null` | 认证来源标识：`"claim"`=领取流程内认证，`"single"`=单独活体认证页面 |
| 8 | `createTime` | date | 是 | 当前时间 | 本次活体认证完成的时间 |

**索引建议**：

| 索引字段 | 类型 | 说明 |
|----------|------|------|
| `_openid` + `createTime` | 复合降序索引 | 按用户查询活体认证历史 |

**枚举值定义**：

| 字段 | 值 | 中文含义 |
|------|----|----------|
| `source` | `"claim"` | 流程内认证 — 在领取任务的认证步骤中完成 |

### 2.6 `agreements` — 协议模板表

> **用途**：存储协议模板内容，支持版本管理和生效控制。  
> **权限**：所有用户可读，仅后台可写。  
> **关联**：← `agreement_signatures.agreementId`

| # | 字段名 | 类型 | 必填 | 默认值 | 中文注释 |
|---|--------|------|:----:|--------|----------|
| 1 | `_id` | string | 是 | 自动 | 协议模板ID（云数据库自动生成） |
| 2 | `title` | string | 是 | — | 协议标题（如"任务领取服务协议"） |
| 3 | `content` | string | 是 | — | 协议正文（富文本HTML格式，支持标题、段落、列表等排版） |
| 4 | `version` | string | 是 | — | 协议版本号（语义化版本，如 v1.0.0、v1.1.0） |
| 5 | `isActive` | boolean | 是 | `false` | 是否为当前生效版本（true=当前生效 / false=历史版本），同一时间仅有一条为true |
| 6 | `effectiveDate` | date | 否 | `null` | 该版本协议正式生效的日期 |
| 7 | `createTime` | date | 是 | 当前时间 | 协议模板创建时间 |
| 8 | `updateTime` | date | 是 | 当前时间 | 协议模板最近更新时间 |

**索引建议**：

| 索引字段 | 类型 | 说明 |
|----------|------|------|
| `isActive` | 普通索引 | 快速定位当前生效协议 |

**业务规则**：
- 同时只有一条协议 `isActive = true`
- 发布新版本时：旧版本 `isActive→false`，新版本 `isActive→true`
- 签署时带版本号快照，已签署协议不受版本更新影响

**示例数据**：
```json

  "version": "v1.0.0",
  "isActive": true,
  "effectiveDate": "2026-01-01T00:00:00.000Z",
  "createTime": "2026-01-01T08:00:00.000Z",

}
```

### 2.11 `sms_codes` — 短信验证码表

1. **用途**：存储手机短信验证码，用于手机号登录验证  
2. **权限**：云函数写入，仅云函数读取  
3. **关联**：独立表，通过 `phone` 字段与 `users` 隐式关联  

| 字段名 | 类型 | 必填 | 默认值 | 中文注释 |

| `expired` | boolean | ✅ | `false` | 是否已过期 |

| `expireTime` | date | ✅ | — | 验证码过期时间（创建后 5 分钟） |

## 四、数据安全与权限

| 集合 | 读权限 | 写权限 | 说明 |
|------|:------:|:------:|------|
| `users` | 仅本人 | 仅本人 | 通过 `_openid` 自动隔离 |
| `tasks` | 全部用户 | 仅后台 | 任务为公开数据，手写风险控制 |
| `task_claims` | 仅本人 | 本人(部分字段)+后台 | 用户可更新 `deliveryUrls`、`status`；后台可更新审核结果 |
| `identity_records` | 仅本人 | 系统写入 | 认证记录仅创建者可见 |
| `liveness_records` | 仅本人 | 系统写入 | 认证记录仅创建者可见 |
| `agreements` | 全部用户 | 仅后台 | 协议为公开模板 |
| `agreement_signatures` | 仅本人 | 仅本人 | 签署记录仅创建者可见 |
| `settlements` | 仅本人 | 仅后台 | 结算数据敏感，仅本人可见 |
| `tax_records` | 仅本人 | 仅本人 | 算税历史为个人隐私 |
| `help_articles` | 全部用户 | 仅后台 | 帮助内容公开 |
| `sms_codes` | 仅云函数 | 仅云函数 | 验证码敏感，终端不可直接读写 |

> **注意**：以上权限为逻辑层约定。在小程序中，`_openid` 天然由微信云开发自动注入，写操作需在云函数中校验 `_openid` 权限。

## 五、附录

### A. 通用时间戳约定

所有集合统一使用以下时间字段规范：

| 字段名 | 含义 | 写入时机 |
|--------|------|----------|
| `createTime` | 记录创建时间 | 首次新增时写入，永不变更 |
| `updateTime` | 记录更新时间 | 每次修改时更新为当前时间 |

### B. 金额字段规范

| 规范 | 说明 |
|------|------|
| 单位 | 所有金额字段单位为**元（人民币）** |
| 精度 | 小数点后 2 位（如 `50.00`、`0.50`） |
| 存储类型 | `number`（JSON Number，非字符串） |
| 前端展示 | 使用 `Number(value).toFixed(2)` 格式化 |

### C. 脱敏规则

前端展示时的数据脱敏规范：

| 字段 | 展示格式 | 示例 | 实现 |
|------|---------|------|------|
| 姓名 | 隐藏中间字 | 张三 → 张\*，张三丰 → 张\*丰 | `maskText(name, 'name')` |
| 身份证号 | 前3位\*\*后4位 | 110101199001011234 → 110\*\*\*\*\*\*\*\*\*\*\*1234 | `maskText(idCard, 'idCard')` |
| 手机号 | 前3位\*\*\*\*后4位 | 13800138000 → 138\*\*\*\*8000 | `maskText(phone, 'phone')` |
| 银行卡号 | 前4位 \*\*\*\* 后4位 | 6228480012345678912 → 6228 \*\*\*\* 8912 | `maskText(bankCard, 'bankCard')` |

### D. 初始化脚本

项目内置数据库初始化云函数 `cloudfunctions/initFunctions`，支持：

| 操作 | 说明 |
|------|------|
| `createCollections` | 创建全部 11 个集合 |
| `seedTasks` | 插入 5 条示例任务数据（含4条进行中 + 1条已结束） |
| `seedAgreements` | 插入 1 条当前生效的协议模板 |
| `seedAll` | 一键执行全部操作（推荐）— 含用户、领取、结算、算税、认证记录等 |

**种子数据覆盖：**

| `users` | 1 | 我的页面、个人信息页 |
